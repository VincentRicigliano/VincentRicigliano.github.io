Expression
Evaluator

Language
Python 3 · No dependencies
Concepts
Stack ADT · Shunting-Yard · Postfix
Operations
Infix · Prefix · Postfix

A stack-based expression evaluation and conversion system built from scratch at Cal Poly. Implements three distinct algorithms for handling mathematical expressions in different notations, powered by a custom Stack data structure backed by a Python list.

  • Built a Stack ADT from scratch with O(1) push, pop, peek, and size — used as the core data structure throughout
  • postfix_eval evaluates postfix expressions by pushing operands and applying operators on pop, supporting +, -, *, /, ** (exponent), and bit shifts << and >>
  • infix_to_postfix converts standard math notation to postfix using the shunting-yard algorithm, correctly handling operator precedence and parentheses
  • prefix_to_postfix converts prefix (operator-first) notation to postfix by reversing the token list and using the stack to reconstruct the expression
  • Raises descriptive exceptions for invalid tokens, insufficient operands, division by zero, and illegal bit shift on floats

Data Structures · Cal Poly CSC 202 · 2019

Huffman
Encoding

Language
Python 3 · No dependencies
Concepts
Binary Trees · Greedy Algorithm · Compression
Pipeline
Encode + Decode · Full roundtrip

A from-scratch implementation of Huffman coding — a lossless data compression algorithm — built at Cal Poly as part of CSC 202. Frequent characters get short binary codes, rare characters get long ones, reducing total file size. Includes a full encode and decode pipeline.

  • cnt_freq reads any text file and counts the frequency of all 256 possible ASCII characters
  • create_huff_tree builds a binary tree bottom-up using a greedy algorithm: repeatedly combines the two lowest-frequency nodes into a parent until one root remains
  • create_code recursively traverses the tree, assigning binary codes: left branch adds 0, right branch adds 1 — frequent characters get the shortest paths
  • huffman_encode writes a compressed output file with a frequency header and the full binary-coded text
  • huffman_decode reads the header to reconstruct the tree, then walks it bit by bit to recover the original text exactly
